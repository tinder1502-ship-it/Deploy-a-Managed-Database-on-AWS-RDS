# Sterling Checkout - Infrastructure as Code (AWS RDS)

Репозиторий содержит конфигурационные артефакты автоматического развертывания и обеспечения сетевой безопасности слоя данных для SaaS-платформы **Sterling Checkout**.

## Структура репозитория
* `/main.tf` — описание сетевой топологии VPC, приватных подсетей, KMS-ключей, групп безопасности и инстанса RDS PostgreSQL Multi-AZ.
* `/variables.tf` — конфигурационные переменные среды.
* `/docs/database_requirements_summary.pdf` — краткое описание архитектурных требований к БД.
* `/docs/production_readiness_criteria.pdf` — протокол и чек-лист готовности к промышленному запуску.

## Архитектурные особенности развертывания
1. **Сетевой контур:** Полная изоляция от публичного интернета (`PubliclyAccessible = false`). Размещение исключительно в приватных подсетях `10.0.1.0/24` и `10.0.2.0/24`.
2. **Безопасность данных (At-Rest):** Включено аппаратное шифрование дисков AWS KMS (алгоритм AES-256) с автоматической ротацией ключей.
3. **Безопасность трафика (In-Transit):** Через Parameter Group активирована директива `rds.force_ssl = 1`. Нешифрованные сессии сбрасываются автоматически на стороне СУБД.
4. **Отказоустойчивость:** Топология Multi-AZ обеспечивает синхронную репликацию данных. Время автоматического восстановления (RTO) при аварии узла составляет `< 60 секунд` с гарантией нулевой потери транзакций (`RPO = 0`).
